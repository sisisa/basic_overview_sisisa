## 0. このプロンプト集の思想（最重要）
- 情報収集は 「知りたいこと（疑問）」から始める
- **note記事用の情報収集** と **学習・調査用の情報収集** を分離する
- **一次情報・権威ある出典を優先**し、AIの推測を排除する
- **収集 → 理解 → 自分の言葉 → 振り返り** を順番に回す

---

## 1. 【用途別】情報収集フローの全体像

### A. note記事を書くための情報収集フロー
```
疑問設定
↓
Perplexity（全体像＋一次情報URL）
↓
Consensus（学術的裏付け）
↓
NotebookLM（構造化・比喩）
↓
note執筆
```

### B. 学習・理解を深めるための情報収集フロー
```
疑問設定
↓
Consensus（科学的根拠）
↓
Perplexity（補足・実例）
↓
NotebookLM（ファインマン）
↓
振り返り
```
## 2. 【最初に必ず書く】疑問定義テンプレート（共通）

```text
■ 今回知りたいこと（疑問）
・私は「なぜ / どのように / 本当なのか」を知りたい：
（例：なぜRAGはLLMのハルシネーションを減らせるのか？）

■ この情報を使う目的
・note記事用 / 学習用 / 両方

■ 知りたくないこと（除外条件）
・表面的な解説のみ
・出典が不明なブログや個人意見
```

## 3. 【note記事用】情報収集プロンプト
### 3-1. Perplexity AI

URL: https://www.perplexity.ai/

```text
目的：
note記事を書くために、事実確認可能で信憑性の高い情報のみを収集する。

プロンプト：
「以下の『疑問』を解消するために調査してください。

■ 疑問：
[ここに疑問を貼る]

■ 情報収集の条件：
・一次情報（公式ドキュメント、論文、政府・大学・研究機関、著名企業の技術ブログ）を優先
・個人ブログや意見記事は補足扱い
・すべての主張に出典URLを必ず明示

■ 出力構成：
1. 結論の要約（3行）
2. 背景・前提知識
3. 核心となる論点（3〜5個）
4. note記事で誤解されやすいポイント
5. 引用可能な出典URL一覧

事実と推測を明確に分け、断定は出典がある場合のみ行ってください。」
```

### 3-2. Consensus（学術的裏付け）

URL: https://consensus.app/

```text
目的：
疑問に対するより深い調査を行うために「科学的・学術的根拠」を持たせる。

プロンプト：
「以下の疑問について、査読済み論文に基づく科学的コンセンサスを教えてください。

■ 疑問：
[疑問を貼る]

(English Query)
What is the scientific consensus on [Topic]?
Please cite peer-reviewed studies.

■ 要約観点：
・研究者間で概ね合意されている点
・実験・調査で示された具体的数値や結果
・反対意見や研究上の限界

出典論文のタイトルと発表年を必ず示してください。」
```

## 4. 【学習・理解用】情報整理プロンプト
### 4-1. NotebookLM（構造化＋比喩）

URL: https://notebooklm.google/

```text
目的：
理解を目的として、情報を構造とイメージで定着させる。

前提：
Perplexity・Consensusの結果をソースとしてアップロード済み。

プロンプト：
「アップロードしたソースのみに基づいて説明してください。

Step1：構造的要約
・専門用語を最小限に
・重要ポイントを3つに整理

Step2：比喩による理解
・日常的なもの（料理・道具・スポーツ等）で例える
・ソースのどの要素が、例えのどの部分かを明示

推測や補完説明は行わないでください。」
```

### 4-2. ファインマン・テクニック（理解の穴埋め）

```text
目的：
「分かったつもり」を排除する。

プロンプト：
「これから私が自分の言葉で説明します。
あなたは厳しい家庭教師として、次を行ってください。

1. ソースとの不一致の指摘
2. 抽象的で曖昧な表現の指摘
3. 理解を深めるための質問を1つ

■ 私の説明：
（ここに自分の説明を書く）

※ ソース外の知識は使わないでください。」
```
