## AIインテリジェント検索・学習プロンプト集
1. 【調査・収集】Perplexity AI 用テンプレート
```
目的： 網羅的かつ最新の情報を、信頼できる出典（URL）付きで収集する。
プロンプト： 「[調査したいトピック]について、以下の構成で詳細に調査し、必ず各回答の根拠となる出典URLを明示して報告してください。
概要と背景： そもそも何なのか、なぜ今注目されているのか。
主要な論点： 専門家の間で議論されている3〜5つの核心的要素。
メリットとリスク： 実装・導入・理解する上での利点と注意点。
最新の動向： 直近1〜2年で起きた重要な変化やニュース。
回答は箇条書きを多用し、客観的な事実に基づいて記述してください。」
```

2. 【論文調査】Consensus 用テンプレート
```
目的： 学術的なエビデンス（論文）に基づいた「科学的根拠」を特定する。 ※Consensusは英語の方が精度が高いため、英語併記を推奨します。
プロンプト： 「[調査したい具体的な仮説や仕組み]について、学術論文に基づいた結論を教えてください。 (English Query: What is the scientific consensus on [Topic]? Please cite peer-reviewed studies.)
特に以下の点に注目して要約してください：
コンセンサス（合意）： 科学者の間で概ね合意されている結論。
具体的な効果・数値： 論文内で示されている具体的なデータ。
研究の限界： まだ解明されていないことや、否定的な研究結果の有無。」
```

3. 【要約・理解】NotebookLM 用テンプレート (Step A)
```
目的： 収集した膨大な情報を、脳に馴染む「例え話」と「構造化」でインプットする。 ※先にPerplexityやConsensusで得た情報をソースとしてアップロードしてから実行してください。
プロンプト： 「アップロードしたソース資料に基づき、以下の2ステップで解説してください。
Step 1：構造的要約 資料の核心部分を、専門用語を最小限にして3つのポイントで要約してください。
Step 2：比喩（例え話）による解説 この概念を全く知らない人でも直感的に理解できるよう、**『日常的なもの（料理、スポーツ、家庭電化製品など）』**に例えて詳しく説明してください。ソース内のどの要素が、例え話のどの部分に対応するかも明示してください。」
```

4. 【ファインマン・テクニック】NotebookLM 用テンプレート (Step B)
```
目的： 自分の言葉で説明し、AIに「理解の穴」を指摘させることで定着を完璧にする。
プロンプト： 「今から私が、この資料の内容を**『ファインマン・テクニック』を用いてあなたに説明します。あなたは『厳しいが親切な家庭教師』**として、私の説明をソース資料と照らし合わせながら、次の3点を評価してください。
正確性のチェック： ソースの内容と矛盾している箇所や、私の誤解はないか。
具体性の欠如： 抽象的な言葉でごまかしている部分（知識の穴）はないか。
理解を深める問い： 私の説明を受けて、ソース資料の核心部分に関する『鋭い質問』を1つ投げてください。
私の説明は以下の通りです： 『（ここに自分の言葉で説明を書く）』」
```

## このテンプレートを運用する際のポイント
1. 「ソースのみ」を徹底する： NotebookLMでは、AIに「ソース設定」がされていることを確認してください。これにより、AIの勝手な想像（ハルシネーション）を防ぎ、信憑性を維持できます。

2. 例え話のズレを確認： 自分が考えた例え話が、実際の理論と少しでもズレていると、AIは容赦なく指摘してくれます。その**「指摘された瞬間」が最も記憶に定着するタイミング**です。

3. 音声解説の活用： 最後にNotebookLMの「Audio Overview（音声概説）」を生成し、自分の理解とAIの要約に乖離がないか耳で確認すると、さらに振り返りの精度が上がります。
